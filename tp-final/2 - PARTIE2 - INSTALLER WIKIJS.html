Installer Wiki.js sur Ubuntu 22.04

Ubuntu 22.04 LTS (iso)
https://releases.ubuntu.com/jammy/ubuntu-22.04.3-desktop-amd64.iso

Cet exercice montre comment installer Wiki.js sur Ubuntu18.04. Wiki et Wikipédia sont deux entités différentes. Un wiki est un logiciel utilisé pour l'écriture et l'édition collaboratives, tandis que Wikipédia est un site Web qui adhère au format Wiki. Il existe de nombreux moteurs Wiki tels que DokuWiki, TiddlyWiki, PmWiki, Wikkawiki etc.

Wiki.js est l'un des moteurs Wiki open source, puissants et modernes basés sur Node.js, Git et Markdown. Il est publié sous la licence publique générale Affero GNU. Wiki.js offre un support pour diverses bases de données telles que MySQL, MariaDB et MSSQL avec le support principal pour PostgreSQL.

Les fonctionnalités intéressantes associées à Wiki.js sont :

  • Il est assez facile à utiliser avec la possibilité de modifier le contenu dans un format Markdown.
  • Prise en charge de contenu multilingue avec prise en charge complète des langues RTL.
  • Prise en charge complète de la sauvegarde et de la synchronisation de contenu pour plusieurs fournisseurs tels que Google Drive, Microsoft OneDrive, AWS S3, Dropbox, Box.net et DigitalOcean.
  • Stockage soutenu par Git.
  • Il a une fonctionnalité de recherche en tant que fonctionnalité intégrée.
  • Gestion intuitive des ressources multimédias - le contenu multimédia peut être inséré dans le contenu.
  • Prend en charge les commentaires.

Installer Git sur Ubuntu 

Wiki.js nécessite Git sur Ubuntu. Ceci peut être réalisé en exécutant la commande :

sudo apt-get install software-properties-common
sudo add-apt-repository -y ppa:git-core/ppa
sudo apt update
apt list --upgradable
sudo apt install git

Une fois installé, vérifiez la version.

git --version

Installer Node.js sur Ubuntu

Pour que Wiki.js s'exécute, nous avons besoin d'installer Node.js. La méthode la plus recommandée consiste à l'installer à partir des référentiels NodeSource.

Activez le référentiel NodeSource sur Ubuntu à l'aide de la commande :

curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -

Avec le référentiel activé, installez le LTS Node.js à l'aide de la commande :

sudo apt-get install -y nodejs

Une fois terminé, vérifiez les versions installées :

node -v && npm -v

Installer et configurer MariaDB sur Ubuntu

Étant donné que la version prise en charge est MariaDB, nous allons installer MariaDB 10.6 sur notre système Ubuntu.

Commencez par ajouter le dépôt :

sudo apt install software-properties-common -y
curl -LsS -O https://downloads.mariadb.com/MariaDB/mariadb_repo_setup
sudo bash mariadb_repo_setup

Maintenant, continuez et installez le serveur de base de données MariaDB.

sudo apt update
apt list --upgradable
sudo apt install mariadb-server mariadb-client

Une fois installé, démarrez et activez MariaDB :

sudo systemctl start mariadb   
sudo systemctl enable mariadb

Renforcez l'instance à l'aide de la commande :

sudo mariadb-secure-installation

CREATE DATABASE wikidb;

GRANT ALL ON wikidb.* TO 'wikidb_user'@'localhost' WITH GRANT OPTION;
FLUSH privileges;
EXIT

Télécharger Wiki.js sur Ubuntu

Pour renforcer la sécurité, nous allons commencer par créer un utilisateur dédié aux processus Wiki.js et Node.js.

L'utilisateur peut être créé avec la commande :

sudo adduser wikijs

Enter new UNIX password: PassW0rd9

Créez un dossier pour le code source wiki.js.

sudo mkdir -p /var/www/wikijs

L'utilisateur créé sera propriétaire du dossier code source.

sudo chown wikijs:wikijs /var/www/wikijs

Passez maintenant à l'utilisateur créé et téléchargez le dernier code source Wiki.js.

sudo su - wikijs
cd /var/www/wikijs
wget https://github.com/Requarks/wiki/releases/latest/download/wiki-js.tar.gz

Une fois le téléchargement terminé, extrayez l'archive.

tar xzf wiki-js.tar.gz
rm wiki-js.tar.gz

Configurer Wiki.js sur Ubuntu

Le fichier de configuration wiki.js sera créé à partir de l'exemple de fichier de configuration comme ci-dessous :

cp config.sample.yml config.yml

Apportez maintenant les ajustements souhaités au fichier de configuration créé :

nano config.yml

Pour accueillir la base de données MariaDB créée, apportez les modifications ci-dessous au fichier :

db:
  type: mariadb
  # PostgreSQL / MySQL / MariaDB / MS SQL Server only:
  host: localhost
  port: 3306
  user: wikidb_user
  pass: PASSWORD
  db: wikidb
  ssl: false
..........
................
# Leave 0.0.0.0 for all interfaces
bindIP: 127.0.0.1

N'oubliez pas de remplacer les informations d'identification correctes de la base de données. Changez également l'adresse IP de liaison en 127.0.0.1 pour des raisons de sécurité.

Une fois les réglages effectués, vérifiez si le serveur fonctionne correctement.

node server

Sortie :

Arrêtez le serveur en cours d'exécution en utilisant CTRL + C . Revenez également à l'utilisateur sudo précédent et continuez avec la configuration.

exit

Créer un service Systemd pour Wiki.js sur Ubuntu

Pour que nous puissions gérer le service wiki.js, nous devons créer un fichier de service systemd.

sudo nano /etc/systemd/system/wikijs.service

Le fichier contiendra les lignes ci-dessous.

[Unit]
Description=Wiki.js
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/node server
Restart=always

User=wikijs
Environment=NODE_ENV=production
WorkingDirectory=/var/www/wikijs

[Install]
WantedBy=multi-user.target

Enregistrez le fichier et rechargez le démon système.

sudo systemctl daemon-reload

Démarrez et activez le service wiki.js :

sudo systemctl start wikijs
sudo systemctl enable wikijs

Vérifiez l'état du service :

systemctl status wikijs

Accéder à l'interface utilisateur Web wiki.js

Continuez et accédez au service Wiki.js à partir du navigateur en utilisant l'URL http://adresse_IP:3000
